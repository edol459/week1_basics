The nav element has no position property therefore it is by default set to static. This means it is just placed in the normal flow of a document.

The nav element's position property is set to fixed, with a top offset of 170px and a left offset of 30px. Since it has no container, the offsets are in relation to the entire webpage. Because it is fixed, the nav bar will stay in that exact position regardless of the user scrolling down or up.

Both articles are placed within the "articles" div class container. The "articles" container is set to relative with a left and top displacement of 30 and 45 pixels. Since the "articles" container is written after the nav bar, it is an element 30px to the right and 45 px below what it's normal webpage flow position would be.

Both of the articles' position properties are set to relative. They both have a displacement of "top: 15px". So the first article's normal position would be in the top left of the "articles" container, but it starts 15 px below that because of the displacement.

The second article's position property is relative. Since it is written after the first article, its normal position would be stacked below the previous article. But it is also given a displacement of top: 15px so it begins 15 px below its normal position- below the first article.

The aside element's position property is set to absolute with left: 50%; right: 10%; and top: 235px; displacements. Absolute positioning implies the displacements are in relation to the element's container, which in this case is the entire webpage. So it is placed 235px below the top of the page, 50% from the left side, and 10% from the right side of the page.

The footer element has a fixed position and is displaced 0px from the bottom of the webpage. The fixed position and displacement implies it will always stay at the bottom of the page regardless of the viewport.
